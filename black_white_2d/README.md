# black_white_2d 

把黑和白的舞蹈拓展到 x、y 两个方向上

## 目前代码

- `concept.pde`：只是简版演示
- `black_white_2d_1.pde`：递增二进制数拼接，作为输入
- `black_white_2d_2.pde`：更多位二进制数
- `black_white_2d_3.pde`：不同进制

## 代码简介

### concept

简版演示：把位置当输入，得到的输出用于画图会怎样？  

或者，在 x、y 两个方向上都放上输入序列  
每个方格取特定位置的输入值  
处理之后按照一定方式显示出来  
会得到什么图形？  

例：  
示例代码里  
`stringX="000001";`  
`stringY="000001";`  
第 1 列第 5 行的方格  
x 方向位置 0  
查找 `stringX` 位置 0 的字符，是 `'0'`，作为 `x1` 的值  
y 方向位置 4  
查找 `stringY` 位置 4 的字符，是 `'0'`，作为 `y1` 的值  
然后计算 `(x1*y1)*255`，得到 0  
`fill(0);` 填充黑色  
于是这个方格涂黑  

第 12 列第 24 行的方格  
x 方向位置 11，大于 `stringX` 的长度 6，除以 6 取余数得到 5  
查找 `stringX` 位置 5 的字符，是 `'1'`，作为 `x1` 的值  
y 方向位置 23，大于 `stringY` 的长度 6，除以 6 取余数得到 5  
查找 `stringY` 位置 5 的字符，是 `'1'`，作为 `y1` 的值  
然后计算 `(x1*y1)*255`，得到 255  
`fill(255);`填充白色  
于是这个方格留白  

### black_white_2d_1

在 `concept.pde` 的基础上  
把输入序列换成 `000001010011100101110111`  
即 `000` 到 `111` 这 8 个递增二进制数的拼接  
看看与 `black_white.pde` 的图形风格的区别

同时尝试不同的处理函数  
有按位与 `&`，按位或 `|`，按位异或 `^`  

### black_white_2d_2

尝试更多位二进制数  

由于数字较大时输入字符串可以长达几百位或更长  
不再直接给出输入字符串  
而是用代码生成  

### black_white_2d_3

尝试不同进制数字  
保留不同数码  
用不同灰色表示  

输入字符串太长  
还是用代码生成  

## 使用方式

用 Processing 打开直接运行。

除了 `concept.pde` 以外，可以通过不同按键看不同处理函数：  
- 数字 `0` 键：按位与
- 数字 `1` 键：按位或
- 数字 `2` 键：按位异或

注意：目前的代码很简陋，**暂不能保存图片**。
