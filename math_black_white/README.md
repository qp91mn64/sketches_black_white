# 数学公式

## 目前代码

- `multiple_1.pde`：$ pn $
- `multiple_2.pde`：$ pn+a_0 $
- `multiple_3.pde`：$ pn+a_0 $，加上第三进制
- `quadratic_1.pde`：$ n*n $
- `quadratic_2.pde`：$ p*n*n $
- `quadratic_3.pde`：$ a_2n^2+a_1n+a_0 $
- `quadratic_4.pde`：$ a_2(n+n_0)^2+a_0 $
- `power_1.pde`：$ n^p $
- `power_2.pde`：$ an^p $
- `power_3.pde`：$ (a_1n+a_0)^p $
- `exponential_1.pde`：$ p^n $

### 整数倍

`black_white.pde` 是一次加 $1$。  
`multiple_1.pde` 则是一次加 $p$，也可以认为是 $p$ 的整数倍取二进制再画图。只需考虑 $p$ 是奇数的情形  
`multiple_2.pde` 加常数项 $a_0$ ，只需考虑 $a_0$ 小于 $p$ 的情形  
`multiple_3.pde` 的运算结果按照第三进制的方式画出  

$p>1$ 或 $a_0>0$ 画出的图形与一般黑和白的舞蹈，即 $p=1$，$a_0=0$ 类似而有所不同  
左侧可能变得复杂或者多重类似图形  
或者再隐藏类似图形  
最左侧则会受常数项影响  

### 二次函数

先计算整数不同形式的二次函数值，再转二进制画图。

只需考虑 $p$ 取奇数情形

可以通过比较 `quadratic_3.pde` 和 `quadratic_4.pde` 的调参方式的不同，比较不同公式形式带来的差异

调用 `java.math.BigInteger` 计算更大的整数以免溢出 

### 幂函数

先计算幂函数值，再转二进制画图。

除了画高次幂以外还可以演示二项式定理

调用 `java.math.BigInteger` 计算更大的整数以免溢出

### 指数函数

目前只有 `exponential_1.pde`。先计算指数，转二进制，再画图。

数字比较大有可能一列画不下只能画出最后几百位  
为了区分，背景不用白色  

只用考虑 $p$ 不是 $2$ 的幂的情形

能画出另一种比较复杂精致的图形。  
$p$ 取 $2$ 的幂加 $1$ ，或 $2$ 的幂减 $1$ ，能把结果分离出不同部分。

调用 `java.math.BigInteger` 计算更大的整数以免溢出

## 主要可调参数

- `cellWidth`：格子宽度
- `cellHeight`：格子高度
- `maxNumber`：最大数位数
- `distance`：相邻两列左侧之间距离
- `offset`：初始数值，方便看很后面的值。部分代码采用 `BigInteger` 类型，理论上无上限，其余代码采用一般整数，大于 2147483647 溢出
- 画布尺寸 `width` x `height`，默认值：
  - `multiple_1.pde`：400 x 800
  - `multiple_2.pde`：400 x 800
  - `multiple_3.pde`：400 x 400
  - `quadratic_1.pde`：400 x 800
  - `quadratic_2.pde`：400 x 800
  - `quadratic_3.pde`：400 x 800
  - `quadratic_4.pde`：400 x 800
  - `power_1.pde`：800 x 400
  - `power_2.pde`：800 x 400
  - `power_3.pde`：240 x 400
  - `exponential_1.pde`：800 x 240

`multiple_3.pde` 才有的：

- `colorString`：颜色对应的字符串，最左边是最大的数字对应的颜色，最右边是0对应的颜色。

`multiple_3.pde` 的 `base` 由 `colorString.length()` 决定，不用手动设置。

公式里的参数：详见各代码。

### 数学公式涉及的参数

对每个整数，按照一个数学公式，无论是一次函数，二次函数，幂函数，还是指数函数，计算出一个值，再把这个值用于后续处理。不同数学公式，能画出风格迥异的图形。

探索的重点可以放在：
- 一次只改变一个系数的取值，其余参数不变，看画出的图形有什么不同；
- 同时改变多个系数取值，其余参数不变，看画出的图形有什么不同；
- 让系数随机取值，看能画出什么图形；
- 如果发现了一种喜欢的图形，尝试寻找能画出类似图形的系数取值，看一看有什么规律，例如与 $2$ （或 `base` ）的幂的联系；
- 猜测还有什么取值也能画出有类似特点的图形，代入公式画出图形验证一下。猜错了也没关系，这里没有作业，没有考试，只有代码，文本和图形。
- 如果可以，尝试解释为什么这些系数就能画出你喜欢的类型。

不需要把发现的每一个规律都解释清楚，甚至给出严谨的证明。我之前也没有见过这么多种风格特异的图形，数学水平也不足以摸清楚里面每个规律，也不能保证猜测是否正确。

想通过不同参数取值，多画几种好看的图形，需要一定的数学技巧，也不是每种系数取值都能画出好看的图形，有些时候画出的图形看起来有点杂乱无章没有什么规律，不必强求看出规律来，换一个取值即可。

可以结合 `println()` 输出中间结果辅助探索。
