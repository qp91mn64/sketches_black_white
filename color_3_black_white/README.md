# 第三颜色系列

## 目前代码

- `grey.pde`：灰色
- `color_3.pde`：第三颜色
- `color_3_random.pde`：随机第三颜色
- `grey_T.pde`：灰色，转置
- `color_3_T.pde`：第三颜色，转置
- `color_3_random_T.pde`：随机第三颜色，转置

### 灰色

背景颜色，0 对应的颜色，1 对应的颜色是随机的黑白灰  
按下 `1` 键或 `2` 键实现互补的视觉效果  

### 第三颜色

指定进制各数码的值从小到大分别对应从白到黑等间距变深的灰色。  
从三进制开始出现灰色。  
例如，三进制时，三个数码 0，1，2 分别对应 255，127.5，0 三种颜色  
四进制时，四个数码 0，1，2，3 分别对应 255，170，85，0 四种颜色  
而 36 进制时，36 个不同的数码 0，1，2，3，...，36分别对应 255，247.71428，240.42857，233.14285，...，0 一共 36 种不同颜色  

目前没有相应的互补效果  

### 随机第三颜色

各数码值分别对应随机灰色  
背景颜色也是随机灰色  

目前没有相应的互补效果  

### 转置

名称后面有 `_T` 的代码  
画出的图形是没有 `_T` 对应代码的图形的转置  

在原有图形的基础上  
把每个格子的行列下标（横纵坐标）互换  
得到相应的转置图形  

## 主要可调参数

- `cellWidth`：格子宽度
- `cellHeight`：格子高度
- `maxNumber`：最大数位数
- `distance`：相邻两列左侧之间距离，默认等于 `maxNumber`
- `offset`：初始数值，最好小于 2147483647 并拉开距离，以免出现负数
- `base`：进制，支持 2 到 36。太大则十进制
- 画布尺寸 `width` x `height`，默认值：
  - `grey.pde`：400 x 800
  - `grey_T.pde`：800 x 400
  - `color_3.pde`：400 x 800
  - `color_3_T.pde`：500 x 500
  - `color_3_random.pde`：400 x 800
  - `color_3_random_T.pde`：400 x 800

`grey.pde`、`grey_T.pde` 才有的参数：

- `color_0`：0 对应的颜色，默认随机
- `color_1`：1 对应的颜色，默认随机
- `backgroundColor`：背景颜色，默认随机
- `colorString`：颜色对应的字符串，最左边是最大的数字对应的颜色，最右边是 0 对应的颜色。`colorString` 字符数量小于 `base` 会报错。

对 `color_3_random.pde`、`color_3_random_T.pde`，`colorArray` 是随机取值，能通过数字 `1` 键改变取值，也是主要参数之一。

## 可能的使用方式

想看横着画的：名称带 `_T` 的。

想看竖着画的：名称没有 `_T` 的。

点击鼠标左键可以暂停，再点击鼠标左键恢复运行。  
点击鼠标右键保存图形。  
每画满一张图都自动保存一次。默认不清屏。  
在控制台查看保存的图片名称等有关信息。  

要看不同视觉效果可以尝试调整各个主要参数的值，每次调整完，重新运行，才能起作用。

对 `grey.pde`、`grey_T.pde`：

主要展示更换 0 和 1 对应的颜色以及背景颜色的视觉效果。也可以尝试固定 `color_1`、`color_0`、`backgroundColor`，例如画浅灰色背景，1 对应黑色，0 对应白色的视觉效果。

按下数字 `1` 键或数字 `2` 键查看互补效果，即把原来 `colorString` 的 0 换成 1、1 换成 0，或 0、1 对应的颜色交换。比较一下两种方式有没有区别，理论上没区别。

对 `color_3_random.pde`、`color_3_random_T.pde`：

按下数字 `1` 键更换各数码值对应的颜色。
